# Production Surface

Inflow Lab is not a production customer-facing application.

It may be deployed as a controlled demo or test utility, but it is not the system of record for FSMA evidence.

## Active Demo/Test APIs

The simulator exposes API and dashboard surfaces for:

- Starting, stopping, stepping, and resetting deterministic simulations
- Loading demo fixtures
- Generating FSMA lifecycle events
- Viewing lot lineage
- Exporting local FDA CSV and EPCIS scaffold outputs
- Sending mock or explicitly configured live trial payloads to RegEngine
- Running smoke tests against local or shared-demo deployments

## Required Guarantees

- Default delivery mode must be safe for demos and local testing.
- Live delivery must require explicit configuration and confirmation.
- Credentials and API keys must never be returned by API responses or rendered in the dashboard.
- Tenant-scoped demo data must remain isolated when Basic Auth or tenant headers are used.
- Remote/shared-demo deployments must enforce Basic Auth and explicit CORS origins.

## Not Production Evidence

The following are not production evidence controls:

- Local JSONL event logs
- Mock ingest responses
- Demo fixture runs
- Generated sample FDA CSV exports
- EPCIS scaffold exports produced only from simulated data

Production evidence must be generated by RegEngine from tenant-scoped persisted records.

## Promotion Rule

A simulator feature may be used in sales, onboarding, or QA only when it:

- Is deterministic or clearly marked as variable
- Runs without customer secrets by default
- Has smoke-test coverage
- Preserves tenant isolation for shared demos
- Does not imply that simulated data is production compliance evidence
